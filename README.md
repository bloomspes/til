# til

내가 하루하루 배우는 것을 그냥 흘려보내지 않으려고 만든 개인 지식베이스. 수집과 초안은 Claude Code와 Codex에 맡기고, 정리와 판단은 직접 한다.

## 왜 쌓는가

나는 하루에도 수많은 글과 도구와 패턴을 스쳐 지나간다. 그 순간엔 "다시 봐야지" 싶지만 탭이 닫히는 순간 거의 다 사라진다. 1년쯤 지나면 "그때 뭐 읽었더라" 하는 감각만 남는다.

이 레포는 그 감각을 붙잡아 두려고 만들었다. 매일 codex가 트렌드를 모아 `sandbox.md`에 쌓아 두면, 월말에 나는 그것을 훑어 "이번 달에 뭐가 있었는지"를 내 문장으로 다시 쓴다. raw 피드는 시간이 지나면 잊혀도 괜찮다. 내가 남기고 싶은 건 그 위에 내가 직접 쓴 문장이다.

편의상 AI가 정리까지 다 하게 두면 결과물도 또 다른 raw 층이 된다. 정리 단계에서 내가 한 번은 개입해야 실제로 머리에 들어온다고 본다.

## 디렉터리

```
til/
├── sandbox.md         # codex가 매일 쌓는 트렌드 피드
├── summaries/         # 월말에 내가 쓰는 요약
├── notes/             # 번역, 깊은 노트
├── archive/           # 옛 자료 (수정하지 않음)
├── skills/            # Claude Code 스킬
└── update-sandbox.sh  # 일일 수집 스크립트
```

자리마다 갱신 주기가 다르다. `sandbox.md`는 매일, `summaries/`는 월 1회, `notes/`는 수시, `archive/`는 거의 건드리지 않는다. 속도가 다른 내용을 한 공간에 섞지 않으려는 배치다.

파일링 규약과 커밋 규칙은 [AGENTS.md](AGENTS.md)에 정리해 뒀다.

## 어떻게 쓰는가

- **매일**: `./update-sandbox.sh`를 한 번 돌린다. codex가 Hacker News와 GitHub Trending에서 48시간 내 항목을 뽑아 `sandbox.md`에 붙인다. 나는 읽거나 넘긴다.
- **월말**: (Phase 2) `/synthesize-month` 스킬로 그 달 트렌드를 훑고 codex가 초안을 제안하면, 내가 골라 `summaries/YYYY-MM.md`에 직접 쓴다.
- **수시로**: 읽은 글은 `analyze-article` 스킬로 번역해 `notes/translations/`에 쌓는다. 한 주제를 깊게 쓰고 싶을 땐 `notes/`에 직접 마크다운으로 적는다.
- **연말**: 쌓인 12개의 월간 요약을 돌아보면서 올해 나한테 뭐가 중요했는지 한두 단락으로 정리한다.

## 앞으로 할 것

- **Phase 1 — 구조 정리 (완료, 2026-04).** 안 쓰던 실습 디렉터리를 지우고 raw / summary / notes / archive 네 축으로 루트를 정리했다. 지금 이 상태.
- **Phase 2 — 합성 루프 (다음).** `/synthesize-month` 스킬을 작성하고 월말 합성이 실제로 굴러가는지 확인하면서 잡는다.
- **Phase 3 — 일일 캡처 (검증 후).** 트렌드 말고 "오늘 뭐 배웠지"를 한 줄로 빠르게 남기는 스킬. 진짜 필요한지는 한 달쯤 써봐야 알 것 같다.
- **Phase 4 — 인사이트 (자연 발생).** summaries가 몇 달치 쌓이면 주제를 가로지르는 공통점이 보일 것 같다. 그때 `notes/insights/`로 승격한다. 미리 만들어 두지는 않는다.

5년 정도 이대로 굴리면 월간 요약 60개쯤, 인사이트 몇십 개, 번역과 노트가 얼마만큼 쌓일 것 같다. 쓸 만한 자산이 될지는 해봐야 안다.

## 운영 원칙

설계하면서 내가 붙잡고 있는 것들. 지금 기준이고 바뀔 수 있다.

- **매일 하는 동작은 얇게.** 매일 codex 결과물에 태그 달고 분류 검증하기 시작하면 금방 지친다. 머리 쓰는 건 월말처럼 몰아서 할 때만.
- **파일을 날짜별로 쪼개지 않는다.** 쪼개면 Grep은 되지만 "그거 어디서 봤더라"는 못 찾는다. 단일 raw 피드와 월간 요약을 랜드마크로 두는 편이 내 기억 구조와 맞는다.
- **분류와 판단은 내가 한다.** 수집만 AI, 정리는 나. 이게 흐려지면 "내가 정리한 내용"이 아니라 "AI가 정리해 준 걸 스크롤한 기록"이 될 뿐이다.
- **죽은 디렉터리는 빨리 치운다.** `golang/`, `java/`, `node/`를 2년 방치한 게 루트를 계속 지저분하게 했다. `git log`에 남으니까 미련 없이 삭제.
- **구조는 잠정적이다.** 이 README도 1년 뒤엔 꽤 달라져 있을 것 같다. 그때 뭐가 틀렸는지 돌아보려고 지금 적어 둔다.

## 이 레포가 아닌 것

혼동을 줄이려고 같이 적어 둔다.

- **블로그가 아니다.** 독자에게 보여 주려고 다듬는 글이 아니라 나에게 남기는 메모다. 그게 공개돼 있을 뿐.
- **Zettelkasten이 아니다.** 양방향 링크나 atomic note 같은 체계는 쓰지 않는다. 마크다운 파일 몇 개면 충분하다.
- **지식 그래프가 아니다.** 쿼리하려는 게 아니라 나중에 읽으려는 거다.
- **완성되는 것이 아니다.** 끝나는 날은 없고 그냥 두꺼워질 뿐이다.
