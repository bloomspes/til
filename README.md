# til

매일 배우는 걸 그냥 흘려보내지 않으려고 만든 개인 지식베이스. Claude Code와 Codex로 수집·초안을 자동화하고, 정리와 판단은 직접 한다.

## 왜 만드는가

개발자는 하루에도 수많은 글·도구·패턴을 스쳐 지나간다. 그 순간엔 "다시 봐야지" 싶지만 탭 닫히면 거의 다 사라진다. 1년쯤 지나면 "그때 뭐 읽었더라"는 감각만 남는다.

이 레포는 그 감각을 붙잡아 두려는 시도다. 매일 codex가 트렌드를 모아 `sandbox.md`에 쌓고, 월말에 그걸 훑어서 "이번 달에 뭐가 있었는지"를 내 문장으로 요약한다. raw 피드는 시간이 지나 잊혀도 괜찮다. 남아야 하는 건 그 위에 내가 직접 쓴 문장이다.

AI가 정리까지 다 해주게 두면 편하지만 그러면 결과물도 또 하나의 raw 층이 된다. 정리 단계에서 사람이 한 번은 개입해야 실제로 머리에 들어온다.

## 구조

```
til/
├── sandbox.md         # codex가 매일 쌓는 트렌드 피드
├── summaries/         # 월말에 내가 쓰는 요약
├── notes/             # 번역물, 깊은 노트
├── archive/           # 옛 자료 (수정 안 함)
├── skills/            # Claude Code 스킬
└── update-sandbox.sh  # 일일 수집 스크립트
```

파일마다 갱신 주기가 다르다. `sandbox.md`는 매일, `summaries/`는 월 1회, `notes/`는 수시, `archive/`는 거의 안 건드린다. 속도가 다른 내용을 같은 공간에서 섞지 않으려는 분리다.

파일링 규약·커밋 규칙은 [AGENTS.md](AGENTS.md).

## 리듬

- **매일** : `./update-sandbox.sh` 한 번. codex가 Hacker News·GitHub Trending에서 48시간 내 항목을 뽑아 `sandbox.md`에 append. 읽거나 넘긴다.
- **월말** : (Phase 2) `/synthesize-month` 스킬로 그 달 트렌드를 훑고 codex가 초안을 제안하면, 내가 읽고 추려서 `summaries/YYYY-MM.md`에 직접 쓴다.
- **수시로** : 읽은 글은 `analyze-article`로 번역해 `notes/translations/`에. 한 주제를 깊게 쓰고 싶을 땐 `notes/` 아래 직접.
- **연말** : 쌓인 12개 `summaries/`를 돌아보면서 "올해 나한테 뭐가 중요했나"를 한두 단락으로.

## 방향

- **Phase 1 — 구조 정리 (완료, 2026-04).** 안 쓰던 실습 디렉토리를 지우고, raw / summary / notes / archive 네 축으로 루트를 정리. 지금 이 상태.
- **Phase 2 — 합성 루프 (다음).** `/synthesize-month` 스킬 작성. 월말 합성이 실제로 굴러가는지 확인하고 잡는다.
- **Phase 3 — 일일 캡처 (검증 후).** 트렌드 말고 "오늘 뭐 배웠지" 한 줄을 빠르게 찍는 스킬. 진짜 필요한지는 한 달쯤 써봐야 안다.
- **Phase 4 — 인사이트 (자연 발생).** summaries가 몇 달치 쌓이면 주제를 가로지르는 공통점이 보일 거다. 그때 `notes/insights/`로 승격. 미리 만들어두지 않는다.

5년 정도 굴리면 월간 요약 60개쯤, 인사이트 몇십 개, 번역·노트 약간. 어느 정도 쓸 만한 자산이 될지는 해봐야 안다.

## 원칙

설계하면서 확인한 것들. 지금 기준이고, 운영하다 보면 바뀔 수도 있다.

- **매일 하는 동작은 최대한 얇게.** 매일 codex 결과물에 태그 달고 분류 검증하다 보면 금방 지친다. 머리 쓰는 건 월말처럼 몰아서 할 때만.
- **파일을 날짜별로 쪼개지 않는다.** 쪼개면 Grep은 되지만 "그거 어디서 봤더라"는 못 찾는다. 단일 raw 피드 + 월간 요약이 랜드마크 역할을 하게 둔다.
- **분류·판단은 사람이.** 수집만 AI, 정리는 내가. 이게 흐려지면 "내가 정리한 내용"이 아니라 "AI가 정리해준 걸 스크롤한 기록"이 된다.
- **죽은 디렉토리는 빨리 정리.** `golang/`, `java/`, `node/`를 2년 방치해 둔 게 구조를 계속 지저분하게 했다. `git log`에 남으니까 미련 없이 삭제.
- **구조는 잠정적이다.** 이 README도 1년 뒤에 꽤 달라져 있을 것 같다. 그때 뭐가 틀렸는지 보려고 지금 적어 둔다.

## 아닌 것들

혼동을 줄이려고 적어 둠.

- **블로그 아님.** 누가 읽기 좋게 다듬는 글이 아니라 내 메모다. 공개돼 있을 뿐.
- **Zettelkasten 아님.** 양방향 링크, atomic note 같은 체계는 안 쓴다. 마크다운 파일 몇 개면 충분.
- **지식 그래프 아님.** 쿼리하려는 게 아니라 나중에 읽으려는 거다.
- **완성되는 게 아님.** 끝이 없고 그냥 두꺼워진다.
