# til

하루하루 배운 것과 통찰을 장기로 쌓는 개인 지식베이스.

## 왜

개발자의 하루는 수많은 글·도구·패턴을 스쳐 지나간다. 그 순간엔 "다시 봐야지" 싶지만 대부분 사라진다. 이 레포는 그걸 붙잡는 곳이다. 단순 북마크 모음이 아니라, 시간이 지나도 자기 언어로 남아 있는 **지식의 축**을 만드는 것이 목표다. Claude Code와 Codex로 수집·합성을 돕되, 증류와 판단은 사람이 한다.

## 구조

- `sandbox.md` — 매일 `update-sandbox.sh`가 Hacker News·GitHub Trending에서 수집하는 개발자 트렌드 raw 피드. 단일 파일로 계속 누적.
- `summaries/YYYY-MM.md` — 월간 합성 요약. 시간·주제로 과거를 회상하는 랜드마크.
- `notes/` — 수동 작성 노트와 외부 글 번역 등 evergreen 콘텐츠. `notes/translations/`에 `analyze-article` 스킬 산출물이 모인다.
- `archive/` — dormant하지만 보존 가치 있는 과거 기록.

파일링 규약과 커밋 규칙은 [AGENTS.md](AGENTS.md) 참고.

## 방향

- **Phase 1 — 구조 정리 (완료, 2026-04).** 방치된 실습 디렉토리를 정리하고 raw / summary / notes / archive 네 축으로 루트를 재편.
- **Phase 2 — 합성 루프 (예정).** `/synthesize-month` 스킬로 `sandbox.md`에서 월간 요약 초안을 만들고 사용자가 확정. 첫 요약은 `summaries/2026-04.md`부터.
- **Phase 3 — 일일 TIL 캡처 (검증 후).** 트렌드 외에 직접 남기는 짧은 일일 노트의 필요성을 일주일 운영으로 확인한 뒤 결정.

프론트매터·태깅 같은 상위 레이어는 콘텐츠가 몇 달치 쌓인 뒤 실제 패턴을 보고 도입한다. 미리 쪼개지 않는다.

## 회고

- **일일 습관과 인지 작업은 분리한다.** 초기엔 일일 트렌드 수집을 스킬 래퍼로 감쌌지만 결국 껍데기였고 삭제했다. 매일 하는 동작은 가능한 한 얇게, 대신 월간 합성처럼 머리를 쓰는 구간에서만 Claude를 깊게 쓴다.
- **회상은 검색이 아니다.** 트렌드를 날짜별 파일로 쪼개면 Grep은 여전히 되지만 "봤던 그거 어디 있지?"가 무너진다. 단일 raw 피드 + 월간 요약이 사람의 기억 구조(시간 덩어리 + 주제 덩어리)에 더 맞았다.
- **AI에게 분류를 맡기지 않는다.** 수집 단계에서 주제 태깅까지 시키면 매일 codex의 분류를 검증해야 해서 리추얼이 끊긴다. 증류와 범주화는 월 1회 사람이 고개를 끄덕이는 지점에서만 한다.
- **죽은 디렉토리는 빨리 묻는다.** 2022–2023년 실습 코드(golang/java/node)를 2년간 안 건드리면서도 남겨 뒀지만, 지식베이스 레포에 속하지 않는 것이었다. 과거에 붙들리지 말고 지금의 용도에 맞게 루트를 정리하는 편이 낫다.
