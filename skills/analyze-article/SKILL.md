# 글 가져오기 및 문서 생성

URL `$0`에서 글을 가져와 한국어 문서를 생성한다.

## 절차

1. **글 가져오기**: WebFetch 또는 `curl`을 사용하여 원문 전체를 가져온다. 요약이 아닌 전문을 확보해야 한다.
2. **파일명 결정**: `$1`이 주어지면 그것을 사용하고, 없으면 글 제목에서 kebab-case로 생성한다.
3. **문서 작성**: `docs/` 디렉토리에 마크다운 파일을 생성한다.
4. **결과 보고**: 생성된 파일 경로와 구성을 간결하게 알려준다.

## 문서 구조

```markdown
# {한국어 제목}

> 원문: [{원문 제목}]({URL}) - {저자}, {날짜}

## 번역

{원문 전체를 자연스러운 한국어로 번역}
{원문의 소제목(h2, h3 등)은 그대로 유지}

---

## 요약

{3~5문장으로 핵심 내용 요약}

## 분석

### 핵심 포인트

| 포인트 | 설명 |
|--------|------|
| ... | ... |

### 시사점

{글에서 얻을 수 있는 인사이트를 번호 리스트로 정리}
{각 항목은 굵은 제목 + 부연 설명}
```

## 규칙

- 번역은 직역보다 자연스러운 의역을 우선한다.
- 기술 용어는 원어를 병기한다 (예: 행 수준 보안(Row-Level Security)).
- 요약은 사실 위주로, 분석은 비판적 시각을 포함한다.
- 원문의 코드 블록이나 기술적 세부사항은 빠짐없이 포함한다.
