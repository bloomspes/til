# AGENTS.md

## 커밋 메시지 규칙

- 커밋 메시지는 한국어로 작성한다.
- 제목은 변경의 핵심을 간결하게 요약한다.
- 본문은 반드시 작성하며, 무엇을 왜 했는지 의도가 드러나도록 한다.
- 본문은 불릿 리스트 대신 문단 형태로 작성한다.

## 디렉터리 구조

- `sandbox.md` : `update-sandbox.sh`로 매일 자동 수집되는 개발자 트렌드 raw 피드. 파일을 분할하지 않고 단일 파일에 계속 누적한다.
- `summaries/YYYY-MM.md` : 월간 합성 요약. Phase 2의 `/synthesize-month` 스킬이 초안을 만들고 사용자가 확정한다.
- `notes/` : 수동 작성 노트와 외부 글 번역 등 evergreen 콘텐츠. `notes/translations/`에 `analyze-article` 스킬 산출물이 모인다.
- `archive/` : dormant하지만 보존 가치 있는 과거 기록.
- 파일명 슬러그는 영문 kebab-case를 기본으로 하되, 이미 존재하는 한국어 파일명은 이관 시 그대로 유지한다.

## 작업 절차

빠른 실행은 진전이 아니다. 방향이 맞아야 진전이다. 사용자의 지시를 받으면 아래 순서를 지킨다.

- **먼저 읽고 생각한다.** 지시가 들어오면 관련 파일, 최근 대화, 기존 지침을 다시 확인한 뒤 움직인다. "지금 당장 편집부터 해야 할 것 같다"는 충동이 들수록 의심한다.
- **피드백이 같은 지점으로 돌아오면 멈춘다.** 같은 문서를 3번 이상 고쳐야 할 상황이면 한 줄씩 패치하지 않는다. 멈춰서 전체 맥락과 근본 원인을 진단하고, 진단을 먼저 사용자에게 말한다. 그다음에 어느 방향으로 갈지 합의하고 움직인다.
- **의도·비전·정체성이 걸린 문서는 사용자의 입력이 먼저다.** Claude가 초안을 상상해서 쓰지 않는다. 사용자의 계기·관심·목표를 짧은 질문으로 받아 낸 뒤, 그 재료로 쓴다. 메타 스캐폴딩(구조, 규칙, 로드맵)만 Claude가 선제적으로 써도 괜찮다.

## 개인 콘텐츠 작성 원칙

README, 회고, 선언문, 비전 문서 등 "사람이 쓴 것처럼 보여야 하는 글"을 이 레포에 쓸 때 지킨다.

- **주어는 항상 사용자(1인칭 "나")다.** Claude, Codex, 스킬, 자동화는 사용자가 쓰는 도구로만 언급한다. 공동 저자로 프레이밍하지 않는다.
- **사용자의 프레임·언어로 쓴다.** Claude가 떠올린 개념 틀(예: "세 가지 속도의 층", "랜드마크", "Phase 1~N 로드맵")을 사용자의 동기·맥락 없이 그냥 박지 않는다. 필요한 재료가 없으면 먼저 묻는다.
- **템플릿 서사를 피한다.** "매일 스쳐 지나감 → 잊힘 → 붙잡자" 같은 누구나 쓸 법한 보편 서사 대신, 사용자만의 계기·실망·기대·경험이 드러나야 한다.
- **메타가 실내용을 밀어내지 않게 한다.** "이 레포를 어떻게 굴릴지"가 90%를 차지하고 "여기 뭘 쌓을지"가 한 줄도 없으면 잘못 쓴 것이다. 실내용(관심 주제, 파고들 질문, 지금의 호기심)이 최소 한 자리를 차지해야 한다.
- **개인 공간을 프로덕트처럼 쓰지 않는다.** Phase 1~N 로드맵, KPI, 성공 지표 같은 소프트웨어 관용구를 개인 기록 레포의 대문에 옮겨 심지 않는다. 메타 계획이 필요하면 AGENTS.md 같은 별도 문서로 분리한다.
- **방어적 섹션을 기본값으로 만들지 않는다.** "이 레포는 X가 아니다" 류 경계 긋기는 개인 공간엔 보통 불필요하다. 명백히 오해될 위험이 있을 때만 쓴다.
- **양적 지표로 비전을 대체하지 않는다.** "X개 요약, Y개 인사이트"보다 "어떤 사람이 되어 있을까"가 진짜 비전이다.
- **아포리즘, 슬로건, 과장된 선언을 쓰지 않는다.** 평범한 대화체, 구체 예시, 솔직한 불확실성으로 쓴다.

이 원칙들은 README를 반복 수정하며 확인된 것이며, 새 패턴이 드러나면 갱신한다.
