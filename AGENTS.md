# AGENTS.md

## 커밋 메시지 규칙

- 커밋 메시지는 한국어로 작성한다.
- 제목은 변경의 핵심을 간결하게 요약한다.
- 본문은 반드시 작성하며, 무엇을 왜 했는지 의도가 드러나도록 한다.
- 본문은 불릿 리스트 대신 문단 형태로 작성한다.

## 파일명 규약

- 파일명 슬러그는 영문 kebab-case를 기본으로 하되, 이미 존재하는 한국어 파일명은 이관 시 그대로 유지한다.

## 작업 절차

사용자의 지시를 받으면 아래 순서를 지킨다.

- **먼저 읽고 생각한다.** 지시가 들어오면 관련 파일, 최근 대화, 기존 지침을 다시 확인한 뒤 움직인다. 바로 편집하고 싶다는 충동이 들수록 한 번 더 의심한다.
- **피드백이 같은 지점으로 돌아오면 멈춘다.** 같은 문서를 세 번 이상 고쳐야 할 상황이면 한 줄씩 패치하지 않는다. 전체 맥락과 근본 원인을 먼저 진단해 사용자에게 말하고, 어느 방향으로 갈지 합의한 뒤 움직인다.
- **의도·비전·정체성이 걸린 문서는 사용자의 입력이 먼저다.** Claude가 초안을 먼저 쓰지 않는다. 사용자의 계기·관심·목표를 짧은 질문으로 받아 낸 뒤 그 재료로 쓴다. 구조·규칙·로드맵 같은 메타 스캐폴딩만 Claude가 선제적으로 써도 된다.
